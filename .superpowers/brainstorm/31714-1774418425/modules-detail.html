🔴 实时刷新 Bug 修复方案
问题根因：page.tsx 中 sseHandlersRegistered.current 守卫导致 handlers 只注册一次，但每轮对话 state 不重置。
修复方式：
  • 每次 handleSendMessage 调用时，重置 contextWindowData / slotDetails 为初始空值
  • 后端确保每轮请求都发送 session_metadata、context_window、slot_details 事件
  • SSE handlers 保持单次注册不变，但 store reset 保证旧数据不残留
模块一：会话元数据与 Token 统计
左列（静态 + session_metadata 事件）
  • 会话名称 ← session_metadata.session_name
  • 上下文限制 ← budget.model_context_window
  • 总 token ← budget.usage.total_used
  • 输入 token ← budget.usage.input_budget
  • 用户消息 ← stateMessages.filter(user)
  • 创建时间 ← session_metadata.created_at
右列（实时更新）
  • 消息数量 ← stateMessages.length
  • 激活模型 ← session_metadata.model
  • 使用率 ← token_update 实时刷新，蓝色高亮
  • 输出 token ← budget.usage.output_reserve
  • 助手消息 ← stateMessages.filter(assistant)
  • 最后活动 ← 每次 done 事件更新时间戳
