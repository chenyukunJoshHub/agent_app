# Tool 系统完整架构 v1 - ASCII 框图

## 完整六层架构（自上而下）

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           Layer 1: Definition Layer                        │
│                              定义层（工具规格）                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                         ToolSpec（工具规格定义）                       │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  必填字段:                                                            │   │
│  │    • name: 工具名称 (如 "send_email")                                 │   │
│  │    • args_schema: 参数 JSON Schema                                    │   │
│  │    • result_schema: 返回值 JSON Schema                                │   │
│  │    • docstring: 工具文档                                              │   │
│  │                                                                       │   │
│  │  安全字段:                                                            │   │
│  │    • effect_class: 副作用级别                                         │   │
│  │      - READ: 无副作用 (搜索/读取)                                      │   │
│  │      - WRITE: 本地写入 (保存文件)                                      │   │
│  │      - EXTERNAL_WRITE: 外部API写 (发邮件/发推文)                       │   │
│  │      - DESTRUCTIVE: 不可逆操作 (删除文件/删表)                          │   │
│  │    • requires_hil: 是否需要人工确认                                    │   │
│  │    • allowed_decisions: 允许的审批动作 [approve/edit/reject]          │   │
│  │                                                                       │   │
│  │  可靠性字段:                                                          │   │
│  │    • timeout: 超时时间 (秒)                                            │   │
│  │    • retry: 重试次数                                                  │   │
│  │    • backoff: 退避因子 (base, max)                                    │   │
│  │    • idempotent: 是否幂等                                             │   │
│  │    • idempotency_key_fn: 自定义幂等键函数                              │   │
│  │                                                                       │   │
│  │  调度字段:                                                            │   │
│  │    • can_parallelize: 能否并行执行                                     │   │
│  │    • concurrency_group: 并发组名 (同组串行，不同组并行)                 │   │
│  │    • depends_on: 依赖的工具名称列表                                    │   │
│  │                                                                       │   │
│  │  治理字段:                                                            │   │
│  │    • permission_key: 权限键 (用于持久化决策)                            │   │
│  │    • pattern_extractor: 参数模式提取器                                 │   │
│  │    • audit_tags: 审计标签                                             │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                      ToolLang（多语言支持）                           │   │
│  │    Python / TypeScript / Go                                          │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                           Layer 2: Registry Layer                          │
│                            注册层（工具注册中心）                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌──────────────────────┐  ┌──────────────────────┐  ┌────────────────────┐│
│  │   Built-in Registry  │  │   Plugin Registry    │  │    MCP Registry    ││
│  │   内置工具注册        │  │   插件工具注册        │  │   MCP工具注册      ││
│  ├──────────────────────┤  ├──────────────────────┤  ├────────────────────┤│
│  │ • 文件读写           │  │ • 动态加载            │  │ • 外部服务桥接     ││
│  │ • Web搜索            │  │ • 版本管理            │  │ • 协议转换         ││
│  │ • 数据库操作         │  │ • 热更新              │  │ • 能力发现         ││
│  │ • 计算工具           │  │ • 依赖隔离            │  │                    ││
│  └──────────────────────┘  └──────────────────────┘  └────────────────────┘│
│                                   ↓                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                     VersionedSnapshot（版本化快照）                   │   │
│  │    • 工具列表版本号                                                   │   │
│  │    • 用于复现与回放                                                   │   │
│  │    • 支持时间点查询                                                   │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                          Layer 3: Management Layer                         │
│                            管理层（策略与审批）                              │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                        ToolManager（工具管理器）                       │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  职责:                                                                │   │
│  │    • 元数据查询：获取工具规格                                         │   │
│  │    • 路由：将调用路由到正确的注册中心                                 │   │
│  │    • 工具发现：扫描和加载工具                                         │   │
│  │    • 版本兼容性检查                                                   │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                   ↓                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                         PolicyEngine（策略引擎）                       │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  决策类型:                                                            │   │
│  │    • ALLOW: 直接执行，无需确认                                        │   │
│  │      - 示例: READ 级别工具、用户已信任的操作                           │   │
│  │                                                                       │   │
│  │    • ASK: 触发 HIL，等待人工确认                                      │   │
│  │      - 示例: EXTERNAL_WRITE 级别、首次执行的写操作                      │   │
│  │                                                                       │   │
│  │    • DENY: 拒绝执行                                                   │   │
│  │      - 示例: 破坏性操作、超出权限范围                                   │   │
│  │                                                                       │   │
│  │  决策依据:                                                            │   │
│  │    • effect_class: 副作用级别                                         │   │
│  │    • pattern 匹配: 参数模式 (如只允许发送到特定域名)                    │   │
│  │    • 用户历史决策: 从 PermissionMemory 读取                            │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                   ↓                                         │
│  ┌──────────────────────┐  ┌─────────────────────────────────────────────┐ │
│  │ ApprovalCoordinator  │←→│         PermissionMemory（权限记忆）          │ │
│  │  审批协调器           │  │         持久化审批决策                        │ │
│  ├──────────────────────┤  ├─────────────────────────────────────────────┤ │
│  │ • HIL 中断管理       │  │ • 用户偏好记录                               │ │
│  │ • approve: 批准      │  │ • 决策时间戳                                 │ │
│  │ • edit: 修改参数     │  │ • TTL 过期管理                               │ │
│  │ • reject: 拒绝       │  │ • 审计日志                                   │ │
│  │ • 恢复协议处理       │  │                                             │ │
│  └──────────────────────┘  └─────────────────────────────────────────────┘ │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                           Layer 4: Execution Layer                         │
│                           执行层（DAG 调度引擎）                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                        Normalizer（规范化器）                          │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  输入: 模型输出的 tool_calls                                          │   │
│  │  输出: 标准化的 ToolCall 对象列表                                      │   │
│  │  处理:                                                                │   │
│  │    • 参数校验与补全                                                   │   │
│  │    • 依赖关系解析 (从 ToolSpec.depends_on)                            │   │
│  │    • 构建依赖图 (DAG)                                                 │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                   ↓                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                        DAGScheduler（DAG 调度器）                      │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  算法: 拓扑排序 (Kahn 算法)                                           │   │
│  │  输出: 执行批次列表                                                   │   │
│  │                                                                       │   │
│  │  示例 DAG:                                                            │   │
│  │                                                                       │   │
│  │      tool_a (无依赖)     tool_b (无依赖)                               │   │
│  │           │                     │                                      │   │
│  │           ↓                     ↓                                      │   │
│  │      tool_c (依赖 a)       tool_d (依赖 b)                             │   │
│  │           │                     │                                      │   │
│  │           └───────→ tool_e ←──────┘ (依赖 c, d)                        │   │
│  │                                                                       │   │
│  │  执行计划:                                                            │   │
│  │    Batch 1: [tool_a, tool_b]  ← 可并行 (无依赖)                       │   │
│  │    Batch 2: [tool_c, tool_d]  ← 可并行 (依赖已满足)                    │   │
│  │    Batch 3: [tool_e]         ← 单独 (等待 c, d 完成)                   │   │
│  │                                                                       │   │
│  │  循环检测: 如果存在循环依赖，抛出异常                                  │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                   ↓                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                       BatchExecutor（批次执行器）                      │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  功能:                                                                │   │
│  │    • 并发控制: 限制并发数 (如最多 10 个工具同时执行)                    │   │
│  │    • 超时管理: 单个工具超时后取消                                      │   │
│  │    • 错误收集: 收集所有失败，不立即中断                                │   │
│  │    • 结果聚合: 等待批次内所有工具完成                                  │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                   ↓                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                       FailureStrategy（失败策略）                      │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  fail-fast (快速失败):                                               │   │
│  │    • 任一工具失败，立即停止所有批次                                   │   │
│  │    • 适用: 强一致性场景 (如金融交易)                                   │   │
│  │                                                                       │   │
│  │  partial-success (部分成功):                                         │   │
│  │    • 记录失败的工具，继续执行后续批次                                 │   │
│  │    • 适用: 容错场景 (如批量通知)                                       │   │
│  │                                                                       │   │
│  │  retry-with-backoff (带退避的重试):                                   │   │
│  │    • 失败后按指数退避重试                                             │   │
│  │    • 适用: 临时性错误 (如网络抖动)                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                        Layer 5: State & Recovery Layer                     │
│                          状态与恢复层（ Exactly-Once ）                      │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                        EventSourcing（事件溯源）                       │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  事件类型:                                                            │   │
│  │    • planned: DAG 调度完成，生成执行计划                              │   │
│  │    • started: 工具开始执行                                            │   │
│  │    • succeeded: 工具执行成功                                          │   │
│  │    • failed: 工具执行失败                                             │   │
│  │    • interrupted: HIL 中断                                            │   │
│  │    • resumed: 从中断恢复                                              │   │
│  │    • skipped: 幂等跳过 (已执行过)                                      │   │
│  │                                                                       │   │
│  │  携带数据:                                                            │   │
│  │    • 事件时间戳                                                       │   │
│  │    • tool_call_id                                                    │   │
│  │    • 批次号                                                           │   │
│  │    • 相关数据 (结果/错误/决策)                                        │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                   ↓                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                       CheckpointStore（检查点存储）                    │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  功能:                                                                │   │
│  │    • 状态快照: 保存当前执行状态                                       │   │
│  │    • 恢复点: 记录可恢复的位置                                         │   │
│  │    • 集成 LangGraph checkpointer                                     │   │
│  │                                                                       │   │
│  │  存储内容:                                                            │   │
│  │    • 当前执行到的批次号                                               │   │
│  │    • 每个工具的执行状态                                               │   │
│  │    • HIL 中断时的上下文                                               │   │
│  │    • 用户的审批决策                                                   │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                   ↓                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                      IdempotencyKeys（幂等键管理）                     │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  目的: 确保 Exactly-Once 语义 (即使恢复也不重复执行)                   │   │
│  │                                                                       │   │
│  │  幂等键生成策略:                                                      │   │
│  │    • 默认: user_id + tool_name + args_hash                           │   │
│  │    • 自定义: 通过 idempotency_key_fn 函数                             │   │
│  │    • 示例: send_email 使用 message_id 作为幂等键                      │   │
│  │                                                                       │   │
│  │  执行流程:                                                            │   │
│  │    1. 执行前检查幂等键是否存在                                        │   │
│  │    2. 如果存在，返回缓存结果 (skipped 事件)                            │   │
│  │    3. 如果不存在，执行工具并保存结果                                  │   │
│  │                                                                       │   │
│  │  示例场景:                                                            │   │
│  │    用户执行 send_email(to="alice@example.com", subject="Hello")       │   │
│  │    → 生成幂等键: "user123:send_email:a3f5b8c2"                        │   │
│  │    → 执行前检查: 键不存在                                              │   │
│  │    → 执行发送邮件                                                     │   │
│  │    → 保存结果到缓存                                                   │   │
│  │    → 中断...                                                          │   │
│  │    → 恢复后再次执行                                                   │   │
│  │    → 检查幂等键: 键已存在                                             │   │
│  │    → 跳过执行，返回缓存结果                                           │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
                                    ↓
┌─────────────────────────────────────────────────────────────────────────────┐
│                        Layer 6: Observability Layer                        │
│                          可观测性层（监控与告警）                            │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                       TraceCollector（追踪收集器）                     │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  追踪内容:                                                            │   │
│  │    • 模型调用 (输入/输出/耗时)                                         │   │
│  │    • 工具调用 (名称/参数/结果)                                         │   │
│  │    • 审批流程 (请求/决策/耗时)                                         │   │
│  │    • 重试行为 (次数/延迟)                                             │   │
│  │    • Context 压缩事件                                                 │   │
│  │                                                                       │   │
│  │  调用链关系:                                                          │   │
│  │    模型调用 → 工具调用 → 子工具调用                                    │   │
│  │    每个节点记录 parent_span_id 建立父子关系                           │   │
│  │                                                                       │   │
│  │  集成:                                                                │   │
│  │    • LangSmith (LangChain 原生)                                       │   │
│  │    • OpenTelemetry (标准)                                             │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                   ↓                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                      MetricCollector（指标收集器）                     │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  关键指标:                                                            │   │
│  │    • tool_success_rate: 工具成功率 (目标 > 95%)                       │   │
│  │    • tool_latency_p50/p99: 延迟分位数                                 │   │
│  │    • approval_latency: 审批耗时 (目标 < 5min)                         │   │
│  │    • parallel_efficiency: 并行效率 (实际/理论时间)                     │   │
│  │    • retry_burn_rate: 重试耗尽次数                                    │   │
│  │    • idempotency_cache_hit: 幂等缓存命中率                            │   │
│  │    • dag_depth: DAG 深度 (批次数)                                     │   │
│  │                                                                       │   │
│  │  聚合维度:                                                            │   │
│  │    • 按工具名称                                                       │   │
│  │    • 按用户                                                           │   │
│  │    • 按时间窗口 (1min/5min/1hour)                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                   ↓                                         │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │                        AlertManager（告警管理器）                      │   │
│  ├─────────────────────────────────────────────────────────────────────┤   │
│  │  告警规则:                                                            │   │
│  │    • tool_success_rate < 0.95 (持续 5min)                             │   │
│  │    • approval_latency > 300s (单次)                                   │   │
│  │    • retry_burn_rate > 10/min                                        │   │
│  │    • parallel_efficiency < 0.7 (持续 10min)                           │   │
│  │                                                                       │   │
│  │  告警渠道:                                                            │   │
│  │    • 日志                                                             │   │
│  │    • Slack / Discord                                                 │   │
│  │    • Email                                                            │   │
│  │    • PagerDuty (严重告警)                                             │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 数据流全景图

```
                                    用户请求
                                        ↓
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 1: Definition Layer                                          │
│  ToolSpec: 定义工具规格 (副作用级别/幂等性/依赖关系)                  │
└─────────────────────────────────────────────────────────────────────┘
                                        ↓
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 2: Registry Layer                                            │
│  从内置/插件/MCP 注册中心加载工具，生成版本化快照                     │
└─────────────────────────────────────────────────────────────────────┘
                                        ↓
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 3: Management Layer                                          │
│                                                                      │
│  ToolManager: 查询工具元数据                                         │
│       ↓                                                              │
│  PolicyEngine: 决策 allow/ask/deny                                  │
│       ├─ allow → 直接继续                                            │
│       ├─ ask → 触发 HIL                                              │
│       └─ deny → 拒绝执行                                            │
│                                                                      │
│  [如果是 ask]                                                        │
│       ↓                                                              │
│  ApprovalCoordinator: 中断执行，等待用户决策                          │
│       ├─ approve → 继续                                              │
│       ├─ edit → 修改参数后继续                                       │
│       └─ reject → 取消                                              │
│       ↓                                                              │
│  PermissionMemory: 保存决策 (下次相同操作自动 allow)                  │
└─────────────────────────────────────────────────────────────────────┘
                                        ↓
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 4: Execution Layer                                           │
│                                                                      │
│  Normalizer: 解析依赖关系，构建 DAG                                   │
│       ↓                                                              │
│  DAGScheduler: 拓扑排序分批                                          │
│       Batch 1: [tool_a, tool_b]  ← 并行                              │
│       Batch 2: [tool_c, tool_d]  ← 并行                              │
│       Batch 3: [tool_e]         ← 单独                               │
│       ↓                                                              │
│  BatchExecutor: 依次执行每个批次                                      │
│       ├─ 并发控制: 限制同时执行的工具数                               │
│       ├─ 超时管理: 单个工具超时取消                                   │
│       └─ 错误收集: 收集失败，不立即中断                               │
│       ↓                                                              │
│  FailureStrategy: 决定失败后行为                                      │
│       ├─ fail-fast: 立即停止所有批次                                  │
│       ├─ partial-success: 继续执行                                    │
│       └─ retry-with-backoff: 重试                                    │
└─────────────────────────────────────────────────────────────────────┘
                                        ↓
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 5: State & Recovery Layer                                    │
│                                                                      │
│  IdempotencyKeys: 检查幂等键                                         │
│       ├─ 已存在 → 跳过，返回缓存 (skipped 事件)                       │
│       └─ 不存在 → 继续执行                                          │
│       ↓                                                              │
│  执行工具                                                           │
│       ↓                                                              │
│  EventSourcing: 记录事件 (started/succeeded/failed)                   │
│       ↓                                                              │
│  CheckpointStore: 保存状态快照                                       │
│       ├─ 支持中断后 resume                                           │
│       └─ 保存幂等键和结果                                            │
└─────────────────────────────────────────────────────────────────────┘
                                        ↓
┌─────────────────────────────────────────────────────────────────────┐
│  Layer 6: Observability Layer                                       │
│                                                                      │
│  TraceCollector: 收集调用链路                                       │
│       ↓                                                              │
│  MetricCollector: 聚合指标                                           │
│       ↓                                                              │
│  AlertManager: 检查告警规则                                          │
└─────────────────────────────────────────────────────────────────────┘
                                        ↓
                                   返回结果
```

## HIL (Human-in-the-Loop) 中断恢复流程

```
正常执行中...
    ↓
PolicyEngine 决策: ask (需要人工确认)
    ↓
ApprovalCoordinator: 触发中断
    ↓
CheckpointStore: 保存当前状态
    • 执行到的批次号
    • 待执行的工具列表
    • 工具参数
    • 上下文信息
    ↓
返回给用户: 等待决策
    ↓
用户决策:
    ├─ approve → 继续
    ├─ edit → 修改参数
    └─ reject → 取消
    ↓
[如果是 edit]
    修改参数
    ↓
恢复执行:
    ↓
CheckpointStore: 加载保存的状态
    ↓
IdempotencyKeys: 检查是否有工具已执行
    ├─ 如果已执行 → 跳过，返回缓存
    └─ 如果未执行 → 继续执行
    ↓
继续执行剩余批次
    ↓
完成
```

## DAG 调度示例

```
原始工具调用:
[
    {name: "validate_email", args: {...}},
    {name: "generate_content", args: {...}},
    {name: "send_email", args: {...}, depends_on: ["validate_email", "generate_content"]},
    {name: "log_sent", args: {...}, depends_on: ["send_email"]}
]

依赖关系:
    validate_email ────────┐
                            ├──→ send_email ──→ log_sent
    generate_content ──────┘

DAGScheduler 拓扑排序:
    Batch 1: [validate_email, generate_content]  ← 可并行 (无依赖)
    Batch 2: [send_email]                       ← 等待 Batch 1 完成
    Batch 3: [log_sent]                         ← 等待 Batch 2 完成

执行时序:
    t0 ──────────────────────────────────────────────────────→ t3
        │         │              │              │
    Batch 1   Batch 1        Batch 2       Batch 3
    (并行)    (完成)         (串行)         (串行)

实际耗时:
    假设每个工具耗时 1s
    • 串行执行: 4s
    • 批次并行: 3s (Batch 1 并行省 1s)
```

## 幂等键示例

```
场景: 发送邮件 (不幂等操作)

工具定义:
    name: "send_email"
    effect_class: EXTERNAL_WRITE
    idempotent: False
    idempotency_key_fn: lambda args: args["message_id"]

执行:
    1. 第一次执行
       参数: {to: "alice@example.com", subject: "Hello", message_id: "msg123"}
       生成幂等键: "user123:send_email:msg123"
       检查缓存: 不存在
       执行发送: ✅ 成功
       保存结果: 到缓存

    2. 中断...

    3. 恢复后再次执行
       参数: {to: "alice@example.com", subject: "Hello", message_id: "msg123"}
       生成幂等键: "user123:send_email:msg123"
       检查缓存: 存在!
       跳过执行: ⏭️ 返回缓存结果
       事件: skipped

    结果: 邮件只发送一次 (Exactly-Once)
```

## 核心设计原则总结

```
1. 契约驱动
   所有工具都有完整的规格定义
   不依赖隐式约定或魔法值

2. 策略与执行分离
   PolicyEngine 只做决策，不执行
   ExecutionLayer 只执行，不决策

3. 批次并行
   不是简单的"全部并行"或"全部串行"
   按依赖关系智能分批
   批次内并行，批次间串行

4. Exactly-Once 语义
   通过幂等键确保写操作只执行一次
   即使中断恢复也不重复

5. 可恢复性
   完整的事件溯源
   检查点快照
   可从任意中断点恢复

6. 可观测性
   统一追踪
   关键指标
   主动告警
```
