# 部署检查清单

## 本地开发环境

### 前置条件
- [ ] Docker 已安装
- [ ] Docker Compose 已安装
- [ ] Python 3.11+ 已安装
- [ ] Node.js 20+ 已安装

### 环境配置
- [ ] 复制 `.env.example` 到 `.env`
- [ ] 更新 `.env` 中的敏感信息
- [ ] 确保 Docker 有足够资源 (4GB+ RAM)

### 启动服务
- [ ] 运行 `./scripts/dev.sh`
- [ ] 访问 http://localhost:3000
- [ ] 访问 http://localhost:8000/docs
- [ ] 检查所有服务健康状态

## 生产环境部署

### 前端 (Vercel)

#### 前置配置
- [ ] 创建 Vercel 账户
- [ ] 连接 GitHub 仓库
- [ ] 获取以下密钥:
  - `VERCEL_TOKEN`
  - `VERCEL_ORG_ID`
  - `VERCEL_PROJECT_ID`

#### 环境变量设置
- [ ] `NEXT_PUBLIC_API_URL`: 生产 API 地址
- [ ] `NEXT_PUBLIC_APP_ENV`: production
- [ ] `NEXT_PUBLIC_SENTRY_DSN`: (可选) Sentry DSN

#### 部署验证
- [ ] 检查部署日志无错误
- [ ] 验证域名 DNS 解析
- [ ] 测试核心功能
- [ ] 检查性能分数 (Lighthouse)

### 后端 (Railway)

- [ ] `DEEPSEEK_API_KEY`: DeepSeek API 密钥
